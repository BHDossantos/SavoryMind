# Phase 07.1 — pytest harness + CI (maintenance slice)

**Status:** done
**Date:** 2026-06-10

## Goal

Pay down the testing debt flagged in every phase summary since Phase 01. Seven phases of features had zero automated tests; every verification was a throwaway smoke script. This slice converts the riskiest paths into a repeatable suite and wires CI.

## Outcome

**33 tests, 4 modules, ~26s runtime**, under `backend/tests/`:

- `conftest.py` — fresh in-memory SQLite per test (`StaticPool`), `get_session` dependency override, API-level factories (`signup`, `make_provider` with service + all-week availability), time helpers (`tomorrow_at`, `days_ahead_at`). Plain `TestClient` (no context manager) so startup hooks — and the APScheduler tick — never fire in tests.
- `test_booking.py` (11) — happy path, double-book 409, overlap 409 + back-to-back OK, past-booking 400, outside-working-hours 409, cancel by customer / provider / stranger-403, cancelled-slot rebookable, deposit booking returns checkout URL + holds slot, stub-confirm flips to paid.
- `test_slot_engine.py` (8) — working-hours bounds, confirmed bookings block, cancelled don't, blocked time excluded, **fresh pending-payment holds + stale releases (TTL)**, no availability → no slots, next-slot uses shortest active service, none without services.
- `test_notifications.py` (6) — booking fan-out (confirmation + 24h + 2h with correct `scheduled_at`), idempotent enqueue, **deposit gating** (nothing until paid), cancellation cancels reminders + enqueues cancellation email, `process_due` sends only due, backdated reminder fires.
- `test_auto_fill.py` (8) — the Phase 07 smoke scenarios as repeatable tests: search logging (auth'd customer only), matching-searchers-only broadcast, canceller excluded, 24h rate limit, past-slot guard, 20-recipient cap, stale-search (beyond 14d lookback) excluded.

**CI:**
- `slotly/.github/workflows/test.yml` — becomes active after migration to `BHDossantos/Slotly` (GitHub only reads root-level workflows).
- `(SavoryMind root)/.github/workflows/slotly-tests.yml` — mirror with a `paths: slotly/**` filter so PRs get CI while the project is parked here. Delete on migration.

**Docs:** README test section; `CONVENTIONS.md` testing section rewritten from "most overdue debt" to actual conventions; CLAUDE.md conventions marker updated.

## Bugs / gotchas found while writing the tests

- **Timing trap:** booking "tomorrow at 10:00" makes its 24h reminder due *immediately* (today 10:00 is already past when running after 10:00 UTC). Correct system behavior, wrong test assumption. Tests asserting on due counts must book ≥3 days out — captured as `days_ahead_at` and documented.
- No product bugs found — the smoke-tested paths held up.
